# Implementation Prompt: Role CRUD System for NPM Package

## Task Overview
Implement a complete CRUD (Create, Read, Update, Delete) system for roles in the NPM package with the following components:
1. **RoleList Component**: Display roles with frontend pagination and delete functionality
2. **RoleForm Component**: Single form component for both creating and updating roles
3. **Integration**: Connect components with the Role API endpoints

---

## API Endpoints Available

### Base URL
All endpoints are prefixed with `/Role`

### Authentication
- **All endpoints require authentication** with `[Authorize]` attribute
- **All endpoints require admin privileges** (user must have `isAdmin: true`)
- Header: `Authorization: Bearer {token}`

## 4. Insert Role

### Endpoint
```
POST /Role/insert
```

### Authentication
- Requires: Bearer token
- Requires: Admin role

### Request Payload Structure
```json
{
  "roleId": 0,
  "slug": "string",
  "name": "string"
}
```

### Request Payload Example
```json
{
  "roleId": 0,
  "slug": "content-creator",
  "name": "Content Creator"
}
```

### Request Payload Notes
- **roleId**: Should be `0` for new roles (will be auto-generated by the backend)
- **slug**: Optional - will be auto-generated from the name if not provided or empty
- **name**: Required - The display name of the role (minimum 1 character)

### Response Structure
```json
{
  "roleId": 0,
  "slug": "string",
  "name": "string"
}
```

### Response Example
```json
{
  "roleId": 6,
  "slug": "content-creator",
  "name": "Content Creator"
}
```

### Validation Rules
1. **name** is required and cannot be empty or whitespace
2. **slug** is auto-generated from name if not provided
3. **slug** must be unique (backend checks for duplicates)
4. **roleId** is ignored for insert operations (always set to 0)

### Response Codes
- `200 OK`: Success - Returns the created role with generated roleId
- `400 Bad Request`: Validation error (e.g., empty name, duplicate slug)
- `401 Unauthorized`: Not authenticated or not an admin
- `500 Internal Server Error`: Server error

### Request Payload Notes
- **roleId**: Required - Must be a valid existing role ID (> 0)
- **slug**: Optional - will be auto-generated from the name if not provided
- **name**: Required - The new display name of the role

### Response Structure
```json
{
  "roleId": 0,
  "slug": "string",
  "name": "string"
}
```

### Response Example
```json
{
  "roleId": 6,
  "slug": "senior-editor",
  "name": "Senior Editor"
}
```

### Validation Rules
1. **roleId** must be greater than 0
2. **name** is required and cannot be empty or whitespace
3. **slug** is auto-generated from name if not provided or empty
4. **slug** must be unique (excluding the current role being updated)
5. Role must exist in the database

### Response Codes
- `200 OK`: Success - Returns the updated role
- `400 Bad Request`: Validation error (e.g., invalid roleId, empty name, duplicate slug)
- `401 Unauthorized`: Not authenticated or not an admin
- `404 Not Found`: Role not found
- `500 Internal Server Error`: Server error

```typescript
interface RoleInfo {
```

```typescript
// Fetch all roles
async function fetchRoles(): Promise<RoleInfo[]> {
  const response = await fetch('/Role/list', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${getAuthToken()}`
    }
  });

  if (!response.ok) {
    if (response.status === 401) {
      throw new Error('Unauthorized - Admin access required');
    }
    throw new Error(`Error ${response.status}: ${response.statusText}`);
  }

  return await response.json();
}

// Delete a role
async function deleteRole(roleId: number): Promise<string> {
  const response = await fetch(`/Role/delete/${roleId}`, {
    method: 'DELETE',
    headers: {
      'Authorization': `Bearer ${getAuthToken()}`
    }
  });

  if (!response.ok) {
    if (response.status === 401) {
      throw new Error('Unauthorized - Admin access required');
    }
    if (response.status === 404) {
      throw new Error('Role not found');
    }
    if (response.status === 400) {
      const error = await response.text();
      throw new Error(error);
    }
    throw new Error(`Error ${response.status}: ${response.statusText}`);
  }

  return await response.json();
}
```

### Frontend Pagination Logic
```typescript
function paginateRoles(
  roles: RoleInfo[], 
  page: number, 
  pageSize: number
): {
  items: RoleInfo[];
  totalPages: number;
  hasPreviousPage: boolean;
  hasNextPage: boolean;
} {
  const startIndex = (page - 1) * pageSize;
  const endIndex = startIndex + pageSize;
  const items = roles.slice(startIndex, endIndex);
  const totalPages = Math.ceil(roles.length / pageSize);

  return {
    items,
    totalPages,
    hasPreviousPage: page > 1,
    hasNextPage: page < totalPages
  };
}
```

## Validation Rules Summary

### Field Validation
1. **Name**
   - Required: Yes
   - Min length: 1 character
   - Max length: No explicit limit (reasonable UI limit recommended)
   - Cannot be empty or whitespace only

2. **Slug**
   - Required: No (auto-generated if not provided)
   - Must be unique across all roles
   - Auto-generated from name by backend
   - Format: lowercase, hyphenated (e.g., "content-manager")

3. **RoleId**
   - For Insert: Must be 0 (will be ignored/auto-generated)
   - For Update: Must be > 0 and exist in database

## Performance Considerations

1. **Frontend Pagination**
   - Only render visible page items
   - Use virtualization for very large lists (100+ items)
   - Debounce search input

2. **API Calls**
   - Cache role list data (invalidate on create/update/delete)
   - Implement retry logic for failed requests
   - Show optimistic updates where appropriate

3. **Form Optimization**
   - Debounce slug generation
   - Validate on blur, not on every keystroke
   - Prevent multiple simultaneous submissions
